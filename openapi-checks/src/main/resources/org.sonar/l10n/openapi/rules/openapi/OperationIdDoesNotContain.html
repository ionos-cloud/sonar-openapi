As defined in the SDK guidelines, the operationId SHOULD not contain both `create` and `post`, or `put` and `update` in name. Having both in the name duplicates without adding information. See https://docs.google.com/document/d/1DZea04mLSH0pqZRrHQxHsOrAxu4J3XfA/edit#heading=h.17dp8vu

Non-compliant Code Example

 "/clusters":
        post:
            operationId: clustersCreatePost

Compliant Solution

 "/clusters":
        post:
            operationId: clustersPost
